## Problem

Once [244](244-command-noop-history-entries.md) routes command paths through the
Edit Baseline, the eager push API — `DocumentHistory::push_document` and
`PixelCanvasHistory::push_snapshot`, plus their wasm-bindgen and UniFFI
surfaces — has **zero production callers**. It stays alive only because the core
tests use it as a setup shortcut.

That leaves a contradiction a reader will trip on: the ADR says the Edit
Baseline is how history gets recorded, while a public FFI method still offers to
record one without any comparison. 244 deletes the *shell-side* wrappers (cheap
— one caller each) but deliberately leaves the core/FFI surface alone to keep
its diff reviewable.

## Solution

Delete the public eager push from both species and both bindings, keeping
`History::push` private to the ring (`end_edit` uses it internally). Migrate the
core history tests — roughly 40 call sites — onto `begin_edit` / `end_edit`
pairs, which has the side benefit of exercising the real recording path instead
of a shortcut no production code uses.

Then, and only then, "the Edit Baseline is the only way to record history" is
true by construction rather than by convention.
