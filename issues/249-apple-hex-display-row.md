## Problem Statement

The web RightPanel shows the foreground color's hex value in a read-only row
between the FG/BG pair and the HSV section (`src/lib/ui-editor/RightPanel.svelte`
`.hex-row`). The Apple shell's color section skips it — the hex value is only
exposed through the foreground swatch's `accessibilityValue`, so sighted users
have no way to read (or communicate) the exact color.

## Solution

Add a hex display row to the Apple `RightPanel` color section, mirroring the
web's placement and look:

- Position: below `fgBgRow`, above the `HSV` section title.
- Content: a tertiary `#` prefix followed by the foreground color's six hex
  digits, uppercase (`Color.hexString` already produces `#RRGGBB` — split off
  the hash for the two-tone styling).
- Style (web `.hex-row`): 28pt control height, 8pt horizontal padding, 4pt gap,
  elevated background, 1pt border, 4pt corner radius; `sm` font for both spans,
  tertiary color for the hash, primary for the digits.
- Read-only display, same as the web — no text input.

## Acceptance criteria

- The hex row renders between the FG/BG pair and the HSV picker, showing the
  current foreground color's uppercase hex digits.
- External foreground changes (palette tap, HSV drag, eyedropper commit, swap)
  update the row live.
- RightPanel snapshot baselines updated on the pinned host.
